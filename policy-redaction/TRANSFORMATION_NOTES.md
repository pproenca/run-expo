## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| `decideActionPolicy` accepts an already-loaded policy object instead of reading `args.actionPolicy` from disk. | File IO belongs in adapters. This module preserves the decision semantics and leaves policy loading to callers. |
| `policyCommand` restores the legacy file-loading command boundary on top of `decideActionPolicy`. | Consumers that need CLI-compatible `policy show/check` output can call the wrapper; tests keep the lower-level policy service reusable. |
| `requireBridgeConfirmation` returns the refusal payload or `null` instead of mutating bridge files. | This module owns the confirmation policy, not bridge file installation. The app bridge SDK owns install-state behavior. |
| Source compiles to ESM `.js` with TypeScript `NodeNext` settings. | Node requires `.js` import specifiers after compilation; source remains `.ts`. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| Full config resolution from flags/project/scratch/defaults. | Config loading is a separate module; this transform only consumes resolved policy/redaction values. |
| Run-record file persistence. | Persistence belongs to the run-record/session module; this transform only preserves summary and sanitization behavior. |
| Bridge file creation/removal. | Covered by `modernized/expo98/bridge-command-adapter`; this package owns only the shared confirmation policy shape. |
| CLI stdout formatting modes. | Output mode selection belongs to the CLI facade; only the shared output-boundary primitive is transformed here. |

## Follow-Ups

- The CLI facade transform should import this module for `policyCommand`, `redactCommand`, policy decisions, redaction, run summaries, and output boundaries.
- `bridge-command-adapter` now preserves bridge confirmation behavior; future package composition should import `requireBridgeConfirmation` directly rather than duplicating the check.
- The diagnostics transform should decide explicitly whether to preserve legacy URL `api_key` behavior or use the stricter reimagined redactor.

## Architecture Review

Initial architecture-critic review found two HIGH gaps:

- RULE-004 global `--allow-runtime-eval` approval was missing.
- RULE-001 policy-denied payload boundary was missing from the API.

Both were fixed in `src/main/policy-service.ts` and covered by characterization tests. The review also flagged missing subprocess truncation coverage for RULE-021; this is now implemented in `truncateSubprocessOutput` and covered by tests.

Remaining LOW note: `redactText` intentionally preserves the narrow legacy semantics for this slice. It only redacts legacy URL query secrets, not arbitrary free-text `password=...` fragments or headers. Callers that need stronger P0 evidence redaction should use the stricter reimagined feedback-loop redactor rather than this equivalence module.
