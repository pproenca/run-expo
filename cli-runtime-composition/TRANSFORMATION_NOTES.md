# cli-runtime-composition Transformation Notes

Generated by `modernize-transform expo98 cli-runtime-composition TypeScript/Node 20 ESM`.

## Scope

This transform adds the final CLI runtime composition boundary that joins the
already transformed parser, argument projector, dispatch envelope, handler
registry, run recorder, output writers, error writer, and facade entrypoint.
It also records the transformed package/export source for each core runtime
component needed by the final executable assembly.

It does not implement command-domain handlers. It preserves the legacy wiring
shape while keeping every side-effecting dependency injectable for the final
executable wrapper.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| This package composes injected functions rather than importing every modernized package directly. | The workspace currently tests packages independently. Injection preserves the legacy wiring contract without forcing local package installation or circular dependencies. |
| Core runtime component sources are recorded as package/export metadata. | This makes final executable assembly auditable while preserving independent package tests and avoiding local package installation in this workspace. |
| `createProcessExitRunner` accepts a `setExitCode` function instead of mutating `process.exitCode`. | The final executable wrapper can pass `process.exitCode = value`; tests remain deterministic and process-global free. |
| Required-handler validation is explicit through `assertRuntimeHasHandlers`. | The legacy object literal failed only when dispatch looked up a missing handler. Composition-time validation catches incomplete final wiring earlier. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| Raw argv parsing, command aliases, output/error envelopes, and argument projection. | Already transformed in `cli-argv-parser`, `command-dispatch-envelope`, and `command-arg-projection`. |
| Concrete command-domain handlers. | Owned by the individual modernized command packages and bound through `tool-handler-registry`. |
| Direct executable shebang and `process.argv.slice(2)` wrapper. | The final executable package should call `createProcessExitRunner(...)` with real process dependencies. |
