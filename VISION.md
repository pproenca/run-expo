# expo98 Vision

`expo98` is a local evidence CLI for Expo and React Native work.
It helps an agent or developer inspect a real app, capture reproducible evidence, and perform safe actions only when policy allows them.

The package is intentionally small: one pnpm-managed npm package, one primary executable, one generated bundled CLI, and a clear publish surface.

## Direction

The current focus is:

- Stable machine-readable CLI output
- Safe defaults for local app, device, bridge, storage, and runtime actions
- Strong secret redaction in output and persisted evidence
- Reliable Expo/React Native inspection across Metro, simulator, device, and app state
- Reproducible evidence for UI refs, screenshots, logs, network, navigation, performance, and runtime diagnostics
- A minimal package that works through `npx expo98 ...` after publish

The next priorities are:

- Better focused tests around high-risk policy and redaction paths
- Better diagnostics when local tooling is missing or stale
- Clearer docs for common Expo and React Native workflows
- Continued cleanup of modernization artifacts so the package stays easy to publish

## Safety Model

`expo98` should make read-only evidence easy and state-changing behavior explicit.

Read commands can run without an action policy. Mutations to apps, devices, bridges, storage, runtime state, clipboard, settings, or similar surfaces must be policy-gated and fail closed when approval is missing.

Evidence should be useful without leaking secrets. JSON output, plain output, errors, run records, URLs, headers, HARs, and summaries must preserve redaction guarantees.

## Package Model

The source of truth is `src/bundled-cli.ts`.
The publishable executable is `cli/expo98.mjs`, generated by `pnpm run build`.

The repository should remain a clean single-package CLI workspace. Runtime dependencies belong in `dependencies` when they are needed by the packed executable; generated caches, analysis trees, package tarballs, and local evidence stay out of git.

## What We Will Not Add For Now

- A monorepo or transformed-module workspace layout
- Hidden startup migrations or silent repair behavior for state-changing paths
- Ungated runtime evaluation, bridge mutation, app mutation, or storage/state writes
- Broad dependencies that are not required by the packed CLI
- A plugin, channel, gateway, or app platform inside this package
- Package publish/version/release automation without an explicit release plan

This is a guardrail, not a permanent limit. Strong evidence and a clear package contract can justify changing direction.
