## Scope

This transform extracts the process-level executable tail from the legacy
`expo-ios` bundle. It bridges process globals to the already transformed CLI
facade/runtime without importing command-domain logic.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| `argv`, `setExitCode`, `writeCliError`, and `exitCodeForError` are injected. | The final executable can pass `process.argv`, `process.exitCode = code`, and the real error helpers; tests stay process-global free. |
| Embedded launchers may pass a custom `argvOffset`. | The default remains the legacy offset `2`, while custom wrappers can avoid re-slicing externally. |
| The wrapper returns the assigned exit code. | This preserves the observable process state while giving tests and embedding callers a direct result. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| CLI argument parsing, command dispatch, output formatting, and command handlers. | Already transformed in parser, facade, dispatch, composition, and command-domain packages. |
| The actual `process.argv`/`process.exitCode` import site. | The final generated executable should provide those globals to `createCliExecutable(...)`. |
