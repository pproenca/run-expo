# app-lifecycle-process-adapter Transformation Notes

Generated by `modernize-transform expo98 app-lifecycle-process-adapter TypeScript/Node 20 ESM`.

## Scope

This transform extracts the concrete legacy subprocess boundary needed by
`app-lifecycle-actions`. The lifecycle package owns command behavior; this
adapter supplies its `execFile` dependency with the legacy `execFilePromise`
semantics from the monolith.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| Node `execFile`, cwd, and env are injected instead of imported directly. | This follows the modernization pattern used by `command-runner-adapter`; final composition can pass `node:child_process.execFile`, `process.cwd`, and `process.env`. |
| This package does not know lifecycle command names or policy decisions. | `app-lifecycle-actions` owns domain behavior and policy gates; this adapter owns process invocation only. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| Crash report filesystem traversal and metadata parsing. | Covered by `app-lifecycle-actions` and `ios-crash-evidence`. |
| Simulator/device resolution. | Covered by iOS simulator/device adapter packages. |
