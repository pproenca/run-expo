# bridge-command-adapter Transformation Notes

Generated by `modernize-transform expo98 bridge-command-adapter TypeScript/Node 20 ESM`.

## Scope

This transform extracts `bridgeCommand(args)` from the legacy CLI. It covers the
command-level action validation, cwd fallback, status and plan envelopes,
delegation to bridge health/domain probing, and confirmation-gated filesystem
mutation for install/remove.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| `bridgeCommand` accepts injected IO and health dependencies. | The default remains filesystem-backed; injected dependencies make mutation, refusal, and delegation behavior deterministic in characterization tests. |
| Generated bridge source fails closed when `__DEV__` is undefined. | This follows the approved RULE-006 hardening already recorded in `bridge-installation`; legacy allowed undefined `__DEV__`, which was identified as a defect. |
| The status, plan, metadata, and source helpers are local to this package rather than imported from sibling packages. | The modernization workspace uses standalone packages without cross-package build wiring. Local copies keep this adapter independently testable while preserving the same contracts. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| Concrete Metro/Hermes bridge health probing. | This command adapter delegates `health`/`domains` to an injected `bridgeHealthPayload`; normalization and health shape are already covered by bridge runtime packages. |
| Final global CLI router wiring. | The existing `tool-handler-registry`, command projection, and facade packages own routing metadata; this slice owns the command behavior. |
