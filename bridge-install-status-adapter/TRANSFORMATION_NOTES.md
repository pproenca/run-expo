# bridge-install-status-adapter Transformation Notes

Generated by `modernize-transform expo98 bridge-install-status-adapter TypeScript/Node 20 ESM`.

## Scope

This transform extracts the filesystem-backed `bridgeInstallStatus(projectRoot)`
adapter from the legacy CLI. The pure status classifier already exists in
`modernized/expo98/bridge-installation`; this package preserves the IO boundary
that reads project files and produces the legacy status envelope.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| `bridgeInstallStatus` accepts optional `readJsonFile`, `pathExists`, and `joinPath` dependencies. | The default export behavior remains filesystem-backed, while injected dependencies make legacy-equivalence cases deterministic and fast. |
| The package duplicates the status envelope instead of importing `bridge-installation` source files. | Existing packages are standalone build units; avoiding cross-package source imports keeps `npm test` per package isolated. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| Bridge install/remove writes in `bridgeCommand`. | Covered by `modernized/expo98/bridge-command-adapter`; this adapter only preserves the status read boundary. |
| Runtime bridge health checks. | Already covered by `bridge-installation` and diagnostics transforms. |
