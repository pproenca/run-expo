## low: install-local resolves CLI path from invocation directory

id: fnd_sig-feat-config-0902ad7be8-14428_a1bc8171a1
category: build-release
confidence: high
triage: risk
status: open
feature: Project config Makefile (feat_config_0902ad7be8)

evidence:
- Makefile:3
- Makefile:8-9

$(abspath cli/expo-ios.mjs) is evaluated relative to the directory where make is invoked, not the directory containing the Makefile. Running this Makefile via -f from another working directory will create or attempt to chmod a path outside the project, so install-local can fail or install a symlink to a nonexistent CLI.

recommendation:
Anchor CLI to the Makefile directory, for example by deriving a Makefile directory variable from `$(lastword $(MAKEFILE_LIST))` and resolving `cli/expo-ios.mjs` relative to it.

test analysis:
No linked tests cover the Makefile or invocation from outside the repository root.

suggested regression test:
Add a smoke test that invokes `make -f path/to/Makefile install-local PREFIX=<tmpdir>` from a different working directory and verifies the installed symlink points at the repo's `cli/expo-ios.mjs`.

minimum fix scope:
Update Makefile path resolution for CLI and add a Makefile install smoke test if the project has an appropriate test harness.

repro:
From the parent directory, run `make -f expo98/Makefile install-local PREFIX=/tmp/expo98-install`. The CLI variable resolves to `<parent>/cli/expo-ios.mjs` rather than `<parent>/expo98/cli/expo-ios.mjs`.
