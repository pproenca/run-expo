## Executive Summary

`expo98` is a **local-first evidence CLI for Expo / React Native iOS work** — it inspects a running app over the Chrome DevTools Protocol (Hermes), drives the iOS simulator via `xcrun`/`simctl`, probes Metro, and captures redacted, reproducible evidence (screenshots, HARs, snapshots, run records) with state-changing actions behind an explicit policy gate. It is **already a modern TypeScript ESM codebase** — Node 20+, pnpm, esbuild, current TS — and is itself the _output of a prior modernization_ that replaced a checked-in monolithic `dist/expo-ios.mjs` with `src/bundled-cli.ts` + a generated bundle. So this is not a "lift a dead language off the mainframe" problem; it is a **technical-debt-and-safety hardening** problem on a young, healthy stack.

Size is moderate: **~26.5K SLOC of hand-written source** (179 files) plus a **17,365-line generated bundle** (`cli/expo98.mjs`) that is deliberately committed. The dominant risks are **not** architectural rot but (1) a **self-inflicted "simulated monorepo"** — 51 single-purpose directories each carrying `src/main/` scaffolding and a dead `node-shims.d.ts`, which VISION.md explicitly says it will _not_ adopt — driving massive copy-paste (the helper `requireString` is redefined **26 times**); and (2) **two documented safety guarantees that the code quietly violates**: the `trace` and `inspector` commands execute arbitrary in-app JavaScript via `Runtime.evaluate` with **no policy gate**, and the generic secret-redactor misses common key names, so secrets can persist to run-records in cleartext.

**Headline recommendation: Refactor (in place).** The stack is current and the public contract is small and well-specified — a rewrite would be waste. A focused ~2–2.5 person-month hardening pass (centralize the policy gate, unify redaction, extract shared utils, add bundle-drift detection, delete dead scaffolding) removes the security gaps and the bulk of the maintenance multiplier without touching the package contract.

## Technical Debt (Top 10, ranked by impact ÷ effort)

1. **Pervasive copy-paste of utility helpers across all 34 command dirs (High).** Same tiny helpers redefined independently: `requireString` ×26, `clampNumber` ×25, `asRecord` ×21, `truncate` ×17, `formatError`/`readJsonFile` ×13, `resolveExpoStateRoot` ×11. 2. **`devtools-diagnostics` re-implements central redaction & CDP logic that already exist as packages (High).** Private `redactValue`/`redactString`/`isSensitiveKey` (`…/index.ts:1135-1167`) shadow `core/policy-redaction`; a local `evaluateHermesExpression` (`:882`) shadows the imported `hermes-cdp-client`. A divergent redactor means a secret can leak from `console`/`errors` output even after the central one is hardened — correctness/safety, not style. → Route through the shared packages; add a redaction test on `console`/`errors`.

1. **`docs/business-rules.md` cites a file that no longer exists.** All 22 rules cite `legacy/expo98/dist/expo-ios.mjs:NNNN` — but `dist/expo-ios.mjs` was removed in the prior modernization (replaced by `cli/expo98.mjs` + `src/`). The docs assert a guarantee the code doesn't fully keep.

5. **The "simulated monorepo" structure is unexplained — and contradicts the stated direction.** A new engineer sees 51 `src/main/` dirs with `node-shims.d.ts` and assumes real pnpm workspaces (`pnpm-workspace.yaml` declares only `packages: ["."]`). ## Recommended Modernization Pattern: **Refactor** (in place)

expo98 does not need rehosting, replatforming, rearchitecting, or rebuilding — those patterns address legacy _runtimes_ or _architectures_, and this system has neither problem. It runs on a current stack (Node 20, TS, ESM, esbuild), exposes a small well-specified public contract (`SPEC.md`), has exactly one runtime dependency, and is _itself_ the clean output of a previous modernization. A **Rebuild** would burn the 75–108 PM replacement cost to recreate code that is already idiomatic.

The actual debt is **internal**: a self-imposed pseudo-monorepo that multiplies copy-paste, a per-command safety model that two commands silently bypass, a redactor with cleartext-leak gaps, and a committed bundle with no drift guard. Every one of these is a **Refactor** — behavior-preserving internal change behind a stable interface — achievable in a focused **~2–2.5 person-month** pass that _raises_ safety (centralized fail-closed gate, unified redaction) while _shrinking_ the maintained surface. Sequence it: (1) characterization tests on policy/redaction + high-CCN commands and the bundle-drift test, (2) centralize the policy gate and unify redaction (the safety wins), (3) extract shared-utils and delete dead scaffolding, (4) decompose god modules. Treat VISION.md's "we will not add a monorepo layout" as the north star — this refactor is largely the act of finally honoring it.
