## Policy

### RULE-001: State-changing commands require explicit action policy
**Category:** Policy
**Priority:** P0
**Source:** `legacy/expo98/dist/expo-ios.mjs:7343-7367`, `legacy/expo98/dist/expo-ios.mjs:6630-6680`
**Plain English:** Any operation that mutates app, device, storage, state, controls, or runtime behavior must be allowed by an explicit action policy.
**Specification:**
  Given a command is read-only
  When  the command is evaluated by the policy engine
  Then  it is allowed without a policy file
  And  the payload records that read actions do not require policy approval

  Given a command has write, device, or runtime side effects
  When  no policy file allows the exact action
  Then  the command returns a policy-denied payload instead of mutating state
  And  the denied payload includes `available: false`, `code: "policy-denied"`, `denied: true`, and the policy decision

  Given a policy file lists the exact action in `allow[]` or `actions[action]`
  When  the state-changing command runs
  Then  the operation is allowed
**Parameters:** Policy action names such as `storage.set`, `state.load`, `controls.press`, and `wait.fn`; allowed values are `allow[]`, `actions[action] === "allow"`, or `actions[action] === true`.
**Edge cases handled:** Missing policy denies mutations; reads always pass; policy path is resolved before loading; denied actions do not call the bridge.
**Confidence:** High - covered by source and tests for storage/state/controls policy gates.

### RULE-019: Wait commands poll on bounded cadence until match or timeout
**Category:** Calculation
**Priority:** P2
**Source:** `legacy/expo98/dist/expo-ios.mjs:1887-1921`, `legacy/expo98/dist/expo-ios.mjs:1972-2050`
**Plain English:** Wait commands repeatedly evaluate predicates on cached refs or runtime state until the predicate matches, fails finally, or reaches timeout.
**Specification:**
  Given a wait predicate is provided
  When  the predicate is not immediately final
  Then  polling repeats at a bounded interval until match or timeout

  Given no predicate is provided
  When  `ms` is greater than zero
  Then  the command sleeps for the clamped duration and reports matched
**Parameters:** `timeoutMs` defaults to `5000` and clamps to `0..60000`; interval is `min(max(floor(timeoutMs / 10), 25), 250)`; sleep `ms` clamps to `0..60000`.
**Edge cases handled:** No snapshot returns unmatched immediately; invalid ref format is final unmatched; stale refs are final unmatched; timeout includes last evidence.
**Confidence:** High - tests cover cached text, visible refs, missing text timeout, and stale refs.

### RULE-020: Ref action points target element centers

**Priority:** P2
**Source:** `legacy/expo98/dist/expo-ios.mjs:2323-2359`, `legacy/expo98/dist/expo-ios.mjs:3085-3090`

  And  point `y` is `box.y + box.height / 2`

### RULE-021: Tool output is truncated with explicit overflow markers
**Category:** Calculation
**Priority:** P2
**Source:** `legacy/expo98/dist/expo-ios.mjs:14`, `legacy/expo98/dist/expo-ios.mjs:12009-12056`
**Plain English:** Large subprocess output is bounded and any truncation is made visible to callers.
**Specification:**

  When  it is formatted

**Edge cases handled:** Nullish values stringify to empty string; subprocess stdout and stderr are attached to formatted error messages before truncation.
**Confidence:** High - source is direct and centralized.
