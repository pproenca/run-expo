## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| Public APIs accept `now` and `randomSuffix` injectors. | The legacy code calls `new Date()` and `Math.random()` directly. Injection keeps production defaults equivalent while making characterization tests deterministic. |
| Session actions are exposed both as focused service functions and a `sessionCommand(args)` wrapper. | Focused services stay reusable for dependent modules; the wrapper now closes the legacy CLI handler gap for final dispatch composition. |
| Filesystem JSON is isolated behind `json-store.ts`. | The legacy bundle writes inline. A tiny store helper removes duplication without hiding behavior or adding a framework. |
| Node built-in type shims are local because this package has no dependency install step. | Existing transformed slices use local shims. This keeps the package self-contained in the modernization workspace. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| Full CLI parsing, aliases, and `writeCliPayload`. | Covered by the CLI facade/dispatch packages; this module now exposes the session handler they can call. |
| Target, snapshot, refs, dashboard, record, and diff artifact flows that read sessions. | Those are separate command/domain transforms; this module only provides the shared session/run-record primitives. |
| Stronger reimagined redaction for arbitrary free text. | This slice preserves legacy run-record redaction exactly. Stronger redaction should be introduced at a hardened evidence boundary with an explicit deviation note. |
