# target-management Transformation Notes

Generated by `modernize-transform expo98 target-management TypeScript/Node 20 ESM`.

## Scope

This transform extracts target discovery, target record construction, target
selection, current-target validation, stale-target reporting, and target
calculation helpers from the legacy CLI. Native simulator discovery and Metro
HTTP transport are represented as injected adapter dependencies.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| Discovery receives adapter functions instead of shelling out to `xcrun` or fetching Metro directly. | Transport belongs to an adapter layer. This module preserves target selection semantics and can be tested deterministically. |
| Session persistence is represented by injected `readLatestSession`, `updateSessionRecord`, `readPersistedTarget`, and `writePersistedTarget`. | The `session-run-records` slice owns session storage. This target module composes with that store rather than duplicating filesystem code. |
| `targetCommand` returns plain payload objects instead of `toolJson(...)`. | CLI output wrapping belongs to the future CLI facade. The payload shapes are equivalent. |
| Malformed Metro array entries are normalized or skipped instead of being blindly trusted. | The legacy bundle trusts Metro JSON. The transformed module preserves normal behavior while maintaining the typed `TargetRecord` contract that downstream snapshot preconditions rely on. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| Native `xcrun simctl list devices` execution. | This is an adapter concern and is represented by `listIosSimulatorTargets`. |
| Metro HTTP `GET /json/list`. | This is an adapter concern and is represented by `fetchMetroTargets`. |
| Android target discovery. | Legacy `discoverTargets` does not populate Android targets; this slice preserves that behavior. Device listing is a separate command/domain. |
| Snapshot precondition handling after target selection. | Snapshot behavior belongs to a future `snapshot-evidence` transform, though it will depend on this module. |

## Architecture Review

The architecture-critic reported no Blockers and two HIGH findings:

| Severity | Finding | Resolution |
| --- | --- | --- |
| High | Unknown Metro payload arrays were cast into trusted domain records. | Added `normalizeMetroTargets` as a runtime boundary parser and characterization coverage for malformed array elements. |
| High | Main API was command-shaped rather than domain-shaped. | Added `listTargets`, `selectTarget`, and `getCurrentTarget`; kept `targetCommand` as the legacy compatibility facade. |

Remaining non-HIGH review notes:

| Severity | Finding | Disposition |
| --- | --- | --- |
| Medium | Discovery and session storage were coupled and could double-read the latest session. | Fixed while addressing the domain API: discovery now accepts `selectedTargetId`; services read session context and pass it in. |
| Medium | Some command tests generated expected targets with `targetRecord`, which can hide regressions in command-level payloads. | Partially accepted: record construction is separately pinned with literal payload tests; future CLI facade tests should use fully literal command-envelope fixtures. |
| Nit | Runtime `action` and `platform` accept broad strings at the compatibility boundary. | Kept for `targetCommand` to mirror CLI input; domain types now include `TargetAction` and `DiscoveryPlatform` for stricter callers. |
