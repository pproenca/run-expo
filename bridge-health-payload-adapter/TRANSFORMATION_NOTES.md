## Scope

This transform extracts `bridgeHealthPayload(args, { action, status, plan })`
from the legacy CLI. It owns the transport decision tree for `bridge health`
and `bridge domains`: install-state short-circuiting, Metro target selection,
Hermes bridge-health expression evaluation, normalized runtime domains, policy
preview, redaction-boundary reporting, and stable unavailable payloads.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| Metro target discovery and Hermes evaluation are injected dependencies. | Existing modernized packages keep network/CDP clients as adapter boundaries; injected dependencies let tests characterize the decision tree without requiring a live Metro runtime. |
| The default Metro dependency returns an unavailable target result instead of opening network sockets. | Standalone package tests must be deterministic. A final composition layer should supply the concrete Metro/Hermes clients. |
| The health expression is represented by the legacy marker expression used at the transport boundary. | Full expression semantics are already covered in `bridge-runtime-contracts`; this package tests that orchestration sends the expected bridge-health expression marker. |
