# bridge-installation Transformation Notes

Generated by `modernize-transform expo98 bridge-installation TypeScript/Node 20 ESM`.

## Scope

This transform extracts bridge metadata, install-state classification, bridge mutation planning, runtime domain normalization, policy preview, and health envelope behavior from the legacy CLI. The target module is pure and IO-free; project file reads/writes and Metro/Hermes probing remain adapter responsibilities.

## Deliberate Deviations

| Deviation | Rationale |
| --- | --- |
| `computeBridgeInstallStatus` accepts `packageJson`, `metadata`, and `sourceExists` instead of reading the filesystem. | File IO is adapter work. The transformed module preserves classification semantics and can be called from real or test adapters. |
| `bridgeMutationRefusal` returns only the refusal payload or `null`. | Actual install/remove file mutations are outside this pure policy/status slice. |
| `shouldRegisterBridge` fails closed when dev mode is undefined. | This is the approved modernization hardening for RULE-006; the generated legacy source text is still preserved by `buildBridgeSource` for equivalence visibility. |
| `bridgeDomainPolicyPreview` keeps an already-resolved action policy path instead of resolving relative paths. | Path resolution belongs to the CLI/config boundary. Absolute paths preserve legacy output exactly; callers can resolve before calling. |

## Not Migrated

| Legacy Area | Reason |
| --- | --- |
| `bridgeCommand` filesystem writes/removes. | Covered by `modernized/expo98/bridge-command-adapter`; this package remains the pure status/planning module. |
| Metro inspector target discovery and Hermes `Runtime.evaluate`. | Transport behavior belongs to a diagnostics/Metro adapter; this module preserves the pure health normalization/envelope logic. |
| App runtime domain execution. | Domain tool calls belong to the app bridge SDK and diagnostics adapters. |

## Follow-Ups

- `bridge-command-adapter` now covers actual filesystem writes and deletes; future package composition should replace its local duplicated helpers with imports from this pure module when cross-package wiring exists.
- `bridge-health-payload-adapter` now covers Metro/Hermes health orchestration; future package composition should replace its local duplicated normalization helpers with imports from this pure module.
- The app bridge SDK should use `bridgeMetadata` as the shared metadata contract.

## Architecture Review

Initial architecture-critic review found one HIGH gap: `buildBridgeSource` still emitted the legacy permissive guard, allowing registration when `__DEV__` was undefined. This was fixed in `src/main/generated-source.ts` so the generated installable source itself returns `development-mode-required` unless development mode is explicit. The characterization test now locks in the hardened generated source behavior.
